Stage 0: HTML and the Web 1.0

The Web is made up of HTML pages

HTML pages are hosted on servers, which your web browser accesses via HTTP requests. Websites contain non-HTML content because their HTML references other types of files (e.g., images).

HTML = Tags + Elements

An HTML document is made out of tags which specify elements. The position of an element is typically defined by the span of its' corresponding start and end tag.

Inline vs. Block

Elements can either be inline or block. They way I think about it, inline elements are analogous to text while block elements define a structural object with properties (which can be modified).

The forest (web) is made of trees (html)

The elements inside an HTML document form a tree-like structure. When looking at HTML code, the tree-like structure should be mirrored in the code's indentation structure. Inline elements are always leaf nodes of the tree, while non-leaf nodes are always block elements.

Lots and lots of boxes

Webpages are made up of a set of nests boxes which reflect the HTML document's tree structure (with each box defined by a block element). When constructing a webpage the first step is to break the page down into the simplest boxes possible and then start building the structure starting with the biggest boxes.

Name your boxes

It is important to provide class names for the elements you create (e.g., <div class="Blah">); and if two elements should be visually consistent, they should generally get the same class name. Along with making your code eaiser to read, names will allow us to easily and consistently update the styling of all elements which are of the same class.

Stage 2: Adding CSS Style to an HTML Document

CSS brings the style

HTML dictates the content of the page while CSS dictates how the page looks. The css is defined in a seperate css file, which is referenced in the heading section of the HTML code.

The debugger is your new besty

The chrome web-browser debugger is a great way to test out CSS changes before committing them to code.

Try not to stutter..

One of the benefits of assigning classes to elements is that it allows you to avoid repetition in your CSS code: the CSS code you write for one class will apply to all instances of the same class. This technique helps keep simliar items visually consistent while also reducing the potential for errors (each line of code has a probability of being wrong!). This also helps when you are trying to borrow concepts from existing pages when building new pages ("How do I make a circle with a number in it? Oh ya, I wrote that .circle{ } css definition back in '11").

CSS will make your head hurt

There is a lot to learn about CSS, but here are some of the key concepts:

  • The size of a block element is defined by the size of three sub-components: content, padding, margin.
  • When writing CSS you need to reference which element you want to style changes to apply to. For example, you can define how you want all elements of a particular class to appear on the screen.
  • Because HTML elements have a tree like structure, CSS definitions can be overlapping. When this happens the browser will choose the most specific definition.
